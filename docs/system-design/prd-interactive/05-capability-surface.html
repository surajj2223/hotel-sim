The symmetry rule

Every endpoint must fully serve ops-web. Every capability notes the ops-web screen it serves and the MCP tool (Wave 2) it backs. No capability may exist solely for the agent.

Authoritative HTTP contract

The authoritative HTTP contract is contracts/WAVE0_02_OPENAPI.yaml. This PRD is the conceptual view — the OpenAPI file has endpoint signatures, DTOs, and error envelopes.

Writes — confirm-then-commit · revalidated · human-gated

All writes re-check availability and price atomically. Repercussive writes require a human-authorisation signal (see HS-06). Conflict on stale state returns 409 with current state.

ops-web has no standalone PRD

ops-web is a complete standalone operations console (Head 1). Its screens are inventoried here, preserving the "no agent-only endpoints" symmetry. Every capability maps to both a UI screen and an MCP tool.

Standing rule

Frozen contracts may not drift silently. Any deviation a builder must make is recorded here with date, the clause affected, what changed, and why. Until a row is arbitrated and the contract re-frozen, the code is the source of truth for that clause.
